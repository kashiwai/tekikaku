🚀 クイックスタート

NET8 SDKを使えば、わずか3行のコードでパチスロゲームを統合できます。

<!-- 1. SDKを読み込み -->
<script src="https://mgg-webservice-production.up.railway.app/sdk/net8-sdk-beta.js"></script>

<script>
  // 2. SDK初期化
  await Net8.init('your_api_key_here');

  // 3. ゲーム作成と開始
  const game = Net8.createGame({
    model: 'milliongod',
    container: '#game-container'
  });

  await game.start();
</script>
✅ これだけで完了！
本番環境のゲームがあなたのサイトに表示されます。

🔧 SDK初期化

1. APIキーの取得

NET8管理画面からAPIキーを取得してください。

環境 APIキープレフィックス 用途
テスト環境 pk_test_xxxxx 開発・テスト用（課金なし）
本番環境 pk_live_xxxxx 本番サービス用（実課金）

2. 初期化オプション

await Net8.init('your_api_key', {
  apiUrl: 'https://mgg-webservice-production.up.railway.app', // デフォルト（省略可）
  debug: true // デバッグモード有効化（オプション）
});
⚠️ セキュリティ注意
APIキーは環境変数で管理し、フロントエンドに直接記述しないでください。
推奨: サーバーサイドで認証後、クライアントに渡す。

🎯 イベントコールバック

ゲームの状態変化をリアルタイムで受け取ることができます。

利用可能なイベント一覧

イベント名 発火タイミング ペイロード例
ready ゲーム準備完了時 { timestamp }
play プレイ開始時（レバー・ボタン操作） { action: 'spin' }
win 当選時 { score: 1000, multiplier: 2 }
lose ハズレ時 { score: 0 }
bonus ボーナス発動時 { type: 'free_spin', count: 10 }
score スコア更新時 { score: 5000 }
end ゲーム終了時 { totalScore: 5000, duration: 300 }
error エラー発生時 { message: 'Error description' }

イベントリスナーの登録

// ゲーム準備完了
game.on('ready', () => {
  console.log('✅ ゲーム準備完了！');
});

// プレイイベント
game.on('play', (data) => {
  console.log('🎮 プレイ:', data);
  // { action: 'spin', eventId: 1, timestamp: 1699999999 }
});

// 勝利イベント
game.on('win', (data) => {
  console.log('🎉 勝利！', data);
  // { score: 1000, multiplier: 2, eventId: 2, timestamp: 1699999999 }
  updateUserBalance(data.score);
});

// ボーナス発動
game.on('bonus', (data) => {
  console.log('⭐ ボーナス発動！', data);
  // { type: 'free_spin', count: 10, eventId: 3, timestamp: 1699999999 }
  showBonusAnimation();
});

// スコア更新
game.on('score', (data) => {
  console.log('📊 スコア更新:', data.score);
  updateScoreDisplay(data.score);
});

// ゲーム終了
game.on('end', (data) => {
  console.log('🏁 ゲーム終了', data);
  // { totalScore: 5000, duration: 300, eventId: 10, timestamp: 1699999999 }
  saveGameResult(data);
});

// エラーハンドリング
game.on('error', (error) => {
  console.error('❌ エラー発生:', error.message);
  showErrorMessage(error.message);
});
💡 ベストプラクティス
- 必ず error イベントを登録してエラーハンドリングを実装してください
- score イベントで残高をリアルタイム更新すると良いUXになります
- end イベントで最終結果をサーバーに保存することを推奨します

🧪 モックモードでのテスト

実機がなくても、モックゲームシミュレーターを使ってSDK統合をテストできます。

モックゲームシミュレーターとは？

🎮 自動プレイモード

ランダムにゲームイベントを発火してコールバック動作を確認

🎯 手動トリガー

好きなタイミングで各イベントを発火してデバッグ

📜 シナリオモード

勝利・ボーナス・敗北などのシナリオを一括実行

📝 イベントログ

送信されたイベントをリアルタイムで確認

モックゲームの使い方

1 モックゲームをiframeで読み込み
<iframe
  id="mock-game"
  src="https://mgg-webservice-production.up.railway.app/sdk/mock-game.html"
  width="100%"
  height="600px"
  frameborder="0"
></iframe>
2 PostMessageでイベントを受信
window.addEventListener('message', (event) => {
  // セキュリティチェック
  if (event.origin !== 'https://mgg-webservice-production.up.railway.app') {
    return;
  }

  const { type, payload } = event.data;

  switch(type) {
    case 'game:play':
      console.log('🎮 プレイイベント:', payload);
      break;
    case 'game:win':
      console.log('🎉 勝利イベント:', payload);
      break;
    // ... 他のイベント
  }
});
3 自動プレイやシナリオでテスト

モックゲーム画面で「自動プレイ」を有効化するか、シナリオボタンをクリックして、イベントコールバックの動作を確認してください。

✅ モックモードの利点
- 実機環境なしで開発・テスト可能
- イベントコールバックの実装確認
- CI/CDパイプラインでの自動テスト
- デモやプレゼンテーションに最適

📦 完全なサンプルコード

実装例: フル機能のゲーム統合

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>NET8 Game Integration</title>
  <style>
    #game-container { width: 100%; height: 800px; }
    .score-display { font-size: 24px; font-weight: bold; }
  </style>
</head>
<body>
  <h1>My Gaming Site</h1>
  <div class="score-display">スコア: <span id="score">0</span></div>
  <div id="game-container"></div>

  <!-- SDK読み込み -->
  <script src="https://mgg-webservice-production.up.railway.app/sdk/net8-sdk-beta.js"></script>

  <script>
    let currentScore = 0;

    async function initGame() {
      try {
        // SDK初期化
        await Net8.init('pk_test_your_api_key');

        // ゲーム作成
        const game = Net8.createGame({
          model: 'milliongod',
          container: '#game-container'
        });

        // イベントリスナー登録
        game.on('ready', () => {
          console.log('✅ ゲーム準備完了');
        });

        game.on('play', (data) => {
          console.log('🎮 プレイ:', data);
        });

        game.on('win', (data) => {
          console.log('🎉 勝利！ スコア:', data.score);
          currentScore += data.score;
          updateScoreDisplay();
        });

        game.on('bonus', (data) => {
          console.log('⭐ ボーナス発動！', data);
          alert(`ボーナス発動！ ${data.type} x${data.count}`);
        });

        game.on('score', (data) => {
          currentScore = data.score;
          updateScoreDisplay();
        });

        game.on('end', (data) => {
          console.log('🏁 ゲーム終了。最終スコア:', data.totalScore);
          // サーバーに結果を保存
          saveGameResult(data);
        });

        game.on('error', (error) => {
          console.error('❌ エラー:', error.message);
          alert('エラーが発生しました: ' + error.message);
        });

        // ゲーム開始
        await game.start();

      } catch (error) {
        console.error('初期化エラー:', error);
      }
    }

    function updateScoreDisplay() {
      document.getElementById('score').textContent = currentScore.toLocaleString();
    }

    function saveGameResult(data) {
      // サーバーAPIに結果を送信
      fetch('/api/save-game-result', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
      });
    }

    // ページ読み込み時に初期化
    window.onload = initGame;
  </script>
</body>
</html>

🔧 トラブルシューティング

よくある問題と解決方法

1. "SDK not initialized" エラー

原因: Net8.init() を呼び出す前にゲームを作成しようとした
解決: 必ず await Net8.init() を先に実行してください

2. "Maximum call stack size exceeded" エラー

原因: SDK読み込みパスが不正、または循環参照が発生
解決: 絶対URLでSDKを読み込んでください
<script src="https://mgg-webservice-production.up.railway.app/sdk/net8-sdk-beta.js">

3. イベントコールバックが発火しない

原因: PostMessageのorigin検証が失敗している可能性
解決: ブラウザのコンソールでエラーを確認してください。
SDKは自動でorigin検証を行います。

4. モックゲームからイベントが届かない

原因: 親ウィンドウのorigin検証でブロックされている
解決: PostMessage受信時に正しいoriginをチェックしてください
if (event.origin !== 'https://mgg-webservice-production.up.railway.app') return;
